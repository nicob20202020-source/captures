# Contexte métier

## Nom du scénario
Ouverture d'une fiche produit 2eme essai

## Identifiant technique
ouverture-dune-fiche-produit-2eme-essai

## Application
PIM

## URL de départ
https://sellapp-preprod.sellbase-plateforme.com/login

## Objectif documentaire
Documenter de manière professionnelle, complète et détaillée une fonctionnalité du PIM à partir d'une session de captures automatisées.

## Public cible
Utilisateurs métier du PIM, administrateurs catalogue, gestionnaires produit, utilisateurs internes ayant besoin d'un guide pas à pas.

## Style attendu
Documentation professionnelle en français, claire, structurée, orientée action, sans jargon technique inutile.

## Consignes de rédaction
- Expliquer ce que l'utilisateur doit faire étape par étape.
- Utiliser le nom métier des écrans, boutons, onglets, tableaux et formulaires.
- S'appuyer sur les captures complètes et les zooms de zones utiles.
- Produire une documentation pédagogique et directement exploitable.
- Ne pas parler de DOM, de sélecteurs ou de détails techniques de capture.
- Utiliser le nom du scénario comme point d'entrée principal pour comprendre la fonction documentée.

## Contexte permanent
- Le projet concerne la documentation complète et détaillée d'un PIM.
- Les sessions successives appartiennent au même univers fonctionnel.
- Les futurs enrichissements doivent rester cohérents avec les documentations précédemment produites.
- Les captures d'écran servent de base d'observation pour comprendre les actions et leurs résultats.

## Remarques
Le nom de scénario fourni au lancement est la principale information variable. Le reste du contexte documentaire est stable d'une session à l'autre.
